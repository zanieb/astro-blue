## Code Blocks in List Items

Lists can contain inline code and full code blocks:

- Install the dependencies:

  ```bash
  npm install @astrojs/starlight
  ```

- Configure your project:

  ```js
  // astro.config.mjs
  import starlight from '@astrojs/starlight';

  export default defineConfig({
    integrations: [starlight({ title: 'My Docs' })],
  });
  ```

- Start the development server:

  ```bash
  npm run dev
  ```

## Nested Lists with Code Blocks

Multiple levels of nesting with code:

1. First step - set up your environment
   - Install Node.js (v18 or higher)
   - Install your preferred package manager:
     ```bash
     npm install -g pnpm
     ```
   - Verify installation:
     ```bash
     pnpm --version
     ```

2. Second step - create your project
   - Run the create command:
     ```bash
     pnpm create astro@latest --template starlight
     ```
   - Navigate to the project directory:
     ```bash
     cd my-docs
     ```

3. Third step - customize your theme
   - Edit the configuration file
   - Add your custom CSS

## Long-Form Content in Lists

Sometimes list items need to contain longer explanations:

1. **Understanding the build process**

   When you run `npm run build`, Astro performs several steps. First, it compiles all your Markdown and MDX files into static HTML. Then, it processes your CSS and bundles any client-side JavaScript. Finally, it generates a sitemap and optimizes images.

   The entire process is designed to produce the smallest possible output while maintaining excellent performance scores.

2. **Configuring for production**

   Production builds require additional configuration. You need to set the correct `site` URL in your Astro config, enable any production-only features like analytics, and ensure environment variables are properly set.

   <Callout type="warning">
     Remember to test your production build locally using `npm run preview` before deploying.
   </Callout>

3. **Deployment options**

   Astro sites can be deployed to virtually any static hosting provider. Popular options include Vercel, Netlify, Cloudflare Pages, and GitHub Pages. Each has its own configuration requirements.

   ```bash
   # Example: Deploy to Vercel
   npx vercel
   ```
